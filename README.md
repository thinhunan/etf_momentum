# ETF 动量策略数据与回测

基于 yfinance 获取 A 股 ETF 池历史行情，本地落库后做**动量 + R² 趋势质量**策略的参数扫描回测。

---

## 回测采用的策略算法

当前回测使用 **ETF 动量 + R² 过滤** 的规则，在池内选趋势强且拟合好的标的持有，定期调仓。

---

## Engine 使用方法（`etf_momentum.py`）

### 1) 生成历史调仓明细（并导出到 `history/{date}/`）

```python
from etf_momentum import EtfMomentum

engine = EtfMomentum(db_dir="db")

# 生成调仓明细，并导出到 history/YYYYMMDD/history_{n}_{r2}_{rebal}_{top_k}.csv
df = engine.back_history(n=5, r2_threshold=0.6, rebal_period=5, top_k=1)

```

```python
signal = engine.next(n=5, r2_threshold=0.6, top_k=1)
```

### 1. 策略思路

- **动量**：用过去一段时间价格走势的“斜率”衡量强弱，斜率大视为动量强。
- **趋势质量**：用线性回归的 **R²** 衡量这段走势是否接近直线、噪音是否小；R² 高才认为是清晰趋势，否则可能是震荡。
- **调仓**：每隔固定交易日（如 5/10/15/20 天）重新算一次全池的斜率和 R²，只保留 R² ≥ 阈值的标的，在其中选斜率最大的若干只等权持有；若无标的通过则空仓。

### 2. 数学形式

对每只 ETF、每个交易日 \(t\)，取过去 **n 个交易日**的收盘价 \(P_{t-n+1}, \ldots, P_t\)：

- 对 **对数价格** 做简单线性回归（时间轴取 0, 1, …, n−1）：
  \[
  \ln P_{t-n+1+j} = a + b \cdot j + \varepsilon_j, \quad j=0,1,\ldots,n-1
  \]
- **斜率 \(b\)**：表示这段窗口内的平均“对数收益/时间”，用作**动量指标**，\(b\) 越大动量越强。
- **R²**：该回归的决定系数，表示价格在这 n 天内有多接近一条直线，用作**趋势质量指标**，R² 越高趋势越清晰。

实现上对全市场、全日期做**滚动窗口**，用解析公式一次性得到每个 \((t, \text{ETF})\) 的 \(b\) 和 R²，不做逐点循环。

### 3. 调仓与持仓规则

| 步骤 | 说明 |
|------|------|
| 调仓日 | 每隔 `rebal_period` 个交易日（如 5/10/15/20 天）执行一次调仓。 |
| 筛选 | 仅保留 **R² ≥ 设定阈值**（如 0.5～0.9）的 ETF。 |
| 排序与选股 | 在通过筛选的标的中，按**斜率 \(b\) 从大到小**排序，取前 **top_k** 只（默认 1 只）。 |
| 持仓 | 在下一个调仓日之前，等权持有这 top_k 只 ETF；若无标的通过筛选则**空仓**（该段收益记为 0）。 |
| 收益计算 | 持仓期内按所选 ETF 的日收益率序列复利得到策略净值。 |

### 4. 参数扫描

回测中对以下三维参数做网格搜索，比较不同组合的**逐年收益、年化收益、最大回撤**等：

- **n**：动量回看天数（如 20, 22, 24, 25, 26, 30）。
- **R² 阈值**：趋势质量门槛（如 0.5, 0.6, 0.7, 0.8, 0.9）。
- **rebal_period**：调仓周期，单位交易日（如 5, 10, 15, 20）。

结果从“每年回报率 + 平均年回报率”等维度对比，并输出热力图、净值曲线和 Top 参数组合。

### 5. 小结

- **信号**：过去 n 日 log 价格线性回归的**斜率**（动量）和 **R²**（趋势质量）。
- **规则**：R² ≥ 阈值 → 按斜率排序 → 取前 top_k 只等权持有，固定周期调仓，无标的则空仓。
- **实现**：滚动窗口向量化计算斜率和 R²，按调仓日应用上述规则并复利得到净值曲线。

## 项目结构

- **etf_pool.csv**：ETF 候选池（代码、名称、类型等）。
- **db/**：每只 ETF 的日线 OHLCV CSV，由 `etf_data_manager` 下载写入。
- **etf_data_manager.py**：基于 yfinance 拉取并落库，支持增量、代理、12 年周期、随机间隔防限流。
- **momentum_backtest.ipynb**：加载 `db/` 数据，运行上述动量+R² 策略的参数扫描回测，输出逐年/年化收益与图表。

## 依赖与运行

- 依赖：`yfinance`、`pandas`、`numpy`、`matplotlib`（见 `requirements.txt`）。
- 下载数据（需先设代理）：  
  `python etf_data_manager.py --proxy http://127.0.0.1:1087`  
  默认 12 年、每只间隔 1～2 秒随机等待。
- 回测：在 Jupyter 中打开 `momentum_backtest.ipynb`，从上到下执行即可。
